## üìä OpenTelemetry Web Instrumentation (Optional)

This project includes optional OpenTelemetry instrumentation for monitoring web performance and user interactions. **Telemetry is disabled by default** to avoid performance overhead.

### Enabling Telemetry

To enable telemetry, set the global flag before initializing your application:

```javascript
// In your index.html or main entry file
window.__ENABLE_TELEMETRY = true;
window.__OTEL_ENDPOINT = 'https://your-otel-collector.example.com/v1/traces'; // Optional
```

Then, initialize telemetry in your application:

```javascript
import { initTelemetry } from './telemetry/otel-web';

// Only runs if window.__ENABLE_TELEMETRY === true
initTelemetry({
  serviceName: 'sentinel-quantum-vanguard',
  serviceVersion: '1.0.0',
  endpoint: 'https://your-otel-collector.example.com/v1/traces', // Optional
});
```

### What Gets Instrumented

When enabled, OpenTelemetry automatically instruments:
- Document load events
- User interactions (clicks, navigation)
- Fetch API requests
- XMLHttpRequest calls

### Configuration

- `serviceName`: Name of your service (default: 'sentinel-quantum-vanguard')
- `serviceVersion`: Version of your service (default: '1.0.0')
- `endpoint`: OTLP trace exporter endpoint (default: 'http://localhost:4318/v1/traces')

## üîß Service Worker & PWA

The application includes a service worker generated by Workbox for offline support and caching. The service worker is automatically generated during the build process:

```bash
npm run build
# Service worker generated at dist/sw.js
```

To manually generate the service worker:

```bash
npm run build:sw
```
