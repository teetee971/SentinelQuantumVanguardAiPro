Souveraineté Numérique

Contrôle des données et indépendance technologique

Principe fondamental

La souveraineté numérique est la capacité pour une organisation ou un État de maîtriser totalement ses données, ses infrastructures numériques et ses choix technologiques, sans dépendance contrainte à des acteurs ou services externes.

Engagement Sentinel

Sentinel Quantum Vanguard AI Pro est conçu dès l'origine pour garantir la souveraineté numérique de ses utilisateurs. Aucune dépendance obligatoire au cloud, aucune collecte opaque, aucun verrouillage technologique (vendor lock-in).

Où vont les données

Données personnelles

Aucune donnée personnelle n'est collectée par Sentinel.

  • Pas de compte utilisateur requis
  • Pas de télémétrie ou analytics
  • Pas de tracking comportemental
  • Pas de cookies tiers
  • Pas de connexion à des services externes non documentés

Données de fonctionnement

Application Web (Console)

Stockage: Navigateur uniquement (localStorage)
Transmission: Aucune
Localisation: 100% local

Application Android

Stockage: Appareil Android uniquement
Transmission: Aucune vers serveurs tiers
Localisation: 100% local

Module Téléphone

Stockage: Local sur appareil
Analyse: Locale (pas d'API externe)
Transmission: Aucune

Sources de données externes

Sentinel agrège des données publiques (OSINT) pour les modules Threat Intelligence et Carte Mondiale :

  • Type: Données publiques uniquement (honeypots, feeds OSINT)
  • Nature: Informations de menaces cyber (IPs, hashes malware, etc.)
  • Traitement: Local, pas de transmission à des tiers
  • Sources: Documentées et transparentes (Abuse.ch, FireHOL, etc.)
Garantie de transparence

Toute connexion réseau effectuée par Sentinel est documentée et vérifiable. Utilisez les DevTools de votre navigateur (onglet Network) pour auditer les communications en temps réel.

Ce qui est local

Traitement 100% local

  • Interface web: Fichiers statiques (HTML/CSS/JS) servis depuis CDN edge, exécution navigateur
  • Application Android: APK autonome, aucun backend requis
  • Module Téléphone: Analyse des numéros en local via bases de données embarquées
  • Préférences utilisateur: Stockées dans localStorage/SharedPreferences localement
  • Logs et journaux: Restent sur l'appareil, pas de transmission automatique

Avantages du traitement local

Vos données ne quittent jamais votre contrôle. Pas de risque de fuite via serveurs tiers.

Performance

Pas de latence réseau. Analyse instantanée sans dépendre d'une connexion Internet.

Disponibilité

Fonctionne hors ligne (offline). Pas de panne due à des serveurs distants.

Souveraineté

Aucune juridiction étrangère impliquée. Données sous loi locale uniquement.

Auto-hébergement possible

Bien que Sentinel soit déployé sur Cloudflare Pages pour la commodité, l'architecture frontend statique permet un auto-hébergement total si nécessaire.

Options de déploiement

Méthode Description Souveraineté
Cloudflare Pages Déploiement officiel actuel (edge-based, global CDN) Dépendance Cloudflare (USA)
Serveur web interne Apache, Nginx, IIS sur infrastructure propre Souveraineté totale
GitHub Pages Hébergement GitHub gratuit (statique) Dépendance GitHub (Microsoft)
Local filesystem Ouverture directe des fichiers HTML dans navigateur Souveraineté absolue
Air-gapped Réseau isolé sans Internet (environnement sensible) Isolation complète

Procédure d'auto-hébergement

  • Étape 1: Cloner le dépôt GitHub : git clone https://github.com/teetee971/SentinelQuantumVanguardAiPro
  • Étape 2: Servir le répertoire public/ via n'importe quel serveur web statique
  • Étape 3: Configurer HTTPS pour sécurité (Let's Encrypt, certificat interne, etc.)
  • Étape 4: Adapter les URLs si nécessaire (chemins absolus → relatifs)
  • Étape 5: Déployer sur infrastructure maîtrisée (datacenter interne, cloud souverain, etc.)
Flexibilité maximale

L'architecture statique de Sentinel garantit qu'aucune modification backend n'est nécessaire pour l'auto-hébergement. Copiez les fichiers, servez-les, c'est tout.

Respect de la souveraineté numérique

Pour les institutions françaises et européennes

  • Conformité RGPD par conception (privacy by design)
  • Aucune transmission de données hors UE non maîtrisée
  • Déployable sur infrastructure souveraine (OVH, Scaleway, etc.)
  • Code source auditable par ANSSI ou organismes équivalents
  • Pas de dépendance à GAFAM (Google, Amazon, Facebook, Apple, Microsoft)

Pour les administrations publiques

  • Aucun transfert de données sensibles vers juridictions étrangères
  • Déploiement possible en réseau isolé (air-gapped) pour secret défense
  • Traçabilité et journalisation complètes (mode institution)
  • Respect des référentiels de sécurité RGS (Référentiel Général de Sécurité)
  • Compatible avec exigences SecNumCloud si hébergé sur cloud qualifié

Pour les entreprises sensibles

  • Aucune fuite de données métier ou stratégiques
  • Déploiement on-premise possible sur datacenter interne
  • Pas de télémétrie ou backdoor
  • Licence open source permettant audit de sécurité interne
  • Indépendance vis-à-vis des éditeurs commerciaux

Conformité réglementaire

RGPD (Règlement Général sur la Protection des Données)

  • Minimisation: Aucune donnée collectée au-delà du strict nécessaire (actuellement: aucune)
  • Finalité: Pas de détournement de finalité (pas de collecte = pas de détournement)
  • Limitation de conservation: Données stockées localement, effacement à discrétion de l'utilisateur
  • Droit d'accès: Utilisateur contrôle 100% de ses données (locales)
  • Portabilité: Données exportables facilement (JSON, CSV selon module)
  • Sécurité: Pas de transmission non chiffrée, pas de stockage centralisé vulnérable

Cloud Act et Extra-territorialité

Le Cloud Act américain permet aux autorités US d'exiger l'accès aux données hébergées chez des fournisseurs américains, même si les données sont stockées hors USA.

Stratégie Sentinel

En ne collectant aucune donnée centralisée et en permettant l'auto-hébergement sur infrastructure souveraine, Sentinel évite totalement cette problématique. Vos données restent sous votre juridiction exclusive.

Garanties et engagements

Engagement de transparence

Nous nous engageons à maintenir Sentinel comme solution respectueuse de la souveraineté numérique. Tout changement affectant ce principe sera annoncé publiquement et clairement.

Ce que nous garantissons

  • Aucune collecte de données personnelles
  • Aucune transmission à des tiers commerciaux
  • Code source ouvert et auditable
  • Architecture déployable en environnement maîtrisé
  • Transparence totale sur les flux de données
  • Respect du RGPD et des réglementations européennes

Ce que nous ne garantissons pas

  • Conformité à des normes spécifiques non auditées (ex: ISO 27001 requiert audit externe)
  • Certification SecNumCloud ou équivalente (nécessite processus de qualification)
  • Support officiel pour environnements classifiés (non testé en conditions secret-défense)
  • Garantie de sécurité absolue (aucun logiciel n'est infaillible)

Vérification et audit

Vous pouvez vérifier nos engagements par vous-même :

Audit technique

  • DevTools navigateur: Ouvrez l'onglet Network et vérifiez les requêtes sortantes
  • Code source: Consultez le dépôt GitHub et analysez le code
  • localStorage: Inspectez le stockage local navigateur (DevTools → Application → Local Storage)
  • APK Android: Décompilez l'APK et auditez les permissions et le code
  • Wireshark/tcpdump: Capturez le trafic réseau et analysez-le

Documentation d'audit

Consultez la documentation d'audit disponible sur GitHub pour une méthodologie complète de vérification de Sentinel.
