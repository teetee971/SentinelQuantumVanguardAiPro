FAQ Technique

Questions techniques fréquentes sur Sentinel Quantum Vanguard AI Pro

1. Est-ce une IA active qui protège réellement mon système ?
NON. Il n'y a AUCUNE IA active dans ce projet.

Réalité technique :

  • C'est une interface web statique (HTML/CSS/JavaScript)
  • Aucun modèle d'IA n'est entraîné ou déployé
  • Aucun traitement en temps réel n'est effectué
  • Les "agents IA" affichés sont des éléments d'interface factices

Pourquoi ? Une vraie IA nécessiterait des serveurs GPU, des modèles entraînés, et une infrastructure backend - rien de tout cela n'existe dans ce projet.

Ce que vous voyez : Une démonstration visuelle de ce à quoi pourrait ressembler une interface d'IA de sécurité.
2. L'application collecte-t-elle mes données personnelles ou surveille-t-elle mon activité ?
NON, absolument ZÉRO collecte de données.

Vérifications techniques :

  • Pas d'analytiques : Pas de Google Analytics, Hotjar, Mixpanel, etc.
  • Pas de cookies : Aucun cookie n'est créé ou lu
  • Pas de stockage local : Pas d'utilisation de localStorage ou sessionStorage
  • Pas d'appels réseau : Ouvrez DevTools → Network, vous verrez ZÉRO requête externe
  • Pas de fingerprinting : Aucune collecte d'empreinte du navigateur

Comment vérifier :

  1. Ouvrez les outils de développement de votre navigateur (F12)
  2. Allez dans l'onglet "Network" / "Réseau"
  3. Rechargez la page
  4. Vous ne verrez que le chargement des fichiers statiques (HTML, CSS, JS) depuis Cloudflare Pages
3. Puis-je remplacer mon antivirus par Sentinel ?
ABSOLUMENT PAS. Sentinel ne remplace PAS un antivirus.

Ce que Sentinel NE FAIT PAS :

  • ❌ Ne scanne pas les fichiers pour détecter des malwares
  • ❌ Ne bloque pas les menaces en temps réel
  • ❌ N'analyse pas le comportement des applications
  • ❌ Ne protège pas contre les virus, trojans, ransomwares
  • ❌ N'a aucune base de données de signatures de malwares
DANGER : Désactiver votre antivirus réel en pensant être protégé par Sentinel vous exposerait à des risques majeurs de sécurité.

Recommandation : Continuez à utiliser un antivirus reconnu (Windows Defender, Kaspersky, Norton, Bitdefender, etc.)

4. Qu'est-ce qui fonctionne réellement dans cette application ?

Fonctionnalités réelles (limitées) :

  • Interface utilisateur : Vous pouvez naviguer dans les différentes sections
  • Affichage de démo : Les logs et statuts affichés sont des exemples pré-définis
  • Responsive design : L'interface s'adapte aux mobiles et tablettes
  • Mode sombre : Thème visuel professionnel fonctionnel

Ce qui ne fonctionne PAS :

  • ❌ Détection de menaces
  • ❌ Analyse du système
  • ❌ Agents IA autonomes
  • ❌ Monitoring réseau
  • ❌ Génération de rapports réels
  • ❌ Intégration avec des systèmes de sécurité
5. Pourquoi créer une interface qui ne fait "rien" ?

Objectifs du projet :

  • Démonstration UI/UX : Montrer à quoi pourrait ressembler une interface de sécurité IA moderne
  • Apprentissage : Projet éducatif pour comprendre les concepts de cybersécurité
  • Prototype : Base de conception pour d'éventuels développements futurs
  • Portfolio : Démonstration de compétences en design d'interface
Transparence : Plutôt que de prétendre avoir des fonctionnalités inexistantes, nous documentons clairement ce qui fonctionne et ce qui ne fonctionne pas.
6. L'application Android est-elle disponible ?
NON, aucune application Android n'existe actuellement.

Statut du pipeline Android :

  • Workflows désactivés : Tous les workflows de build Android sont renommés en .yml.disabled
  • Aucun APK généré : Aucun fichier APK n'a été compilé ou signé
  • Phase A uniquement : Focus actuel sur la console web statique
Pourquoi cette approche ?
  • Éviter de distribuer une application non testée
  • Prévenir les risques de malware ou d'applications dysfonctionnelles
  • Maintenir la transparence sur ce qui existe réellement
7. Quelles permissions l'application demande-t-elle ?

Pour la version web (actuelle) :

  • Aucune permission spéciale requise
  • Fonctionne dans le sandbox sécurisé du navigateur
  • Pas d'accès au système de fichiers
  • Pas d'accès à la caméra ou au microphone
  • Pas d'accès à la localisation

Pour une hypothétique future application Android :

Si une application Android était développée (ce qui n'est pas le cas actuellement), elle demanderait probablement :

  • INTERNET - Pour les requêtes réseau (standard)
  • ACCESS_NETWORK_STATE - Pour vérifier la connexion

Elle ne demanderait PAS :

  • ❌ Accès root
  • ❌ Permissions système dangereuses
  • ❌ Accès aux contacts, SMS, localisation
  • ❌ Accès au stockage interne sans raison

10. Quels sont les risques d'utiliser Sentinel ?

Risques minimaux (interface web) :

  • Pas de risque de malware : Code statique, pas de téléchargement d'exécutables
  • Pas de risque de vol de données : Aucune donnée n'est collectée

11. Que signifie "validation de sécurité" pour l'APK Android ?

Signification technique :

Le message "en cours de validation de sécurité" est une façon transparente de dire :

  • "Nous n'avons pas encore développé d'application Android"
  • "Aucun APK n'existe, donc aucun téléchargement n'est disponible"
  • "Nous préférons ne rien distribuer plutôt que distribuer quelque chose de non testé"

Ce que "validation" impliquerait réellement :

  • Tests de sécurité de l'application
  • Audit de code pour vulnérabilités
  • Signature avec une clé de release (pas la clé debug commitée)
  • Tests sur différents appareils Android
  • Vérification de l'absence de malware
Statut actuel : Aucune de ces étapes n'a été réalisée car aucune application n'existe.
